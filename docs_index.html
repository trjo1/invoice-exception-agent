Run it yourself

The repo is open source under the MIT license. Clone and run locally:

git clone https://github.com/trjo1/invoice-exception-agent.git
cd invoice-exception-agent
uv sync
cp .env.example .env
# add OPENROUTER_API_KEY=sk-or-v1-... to .env
make hitl-serve
# → http://localhost:8080/demo

The first invoice through the pipeline takes ~30-90 seconds (the embedding model loads on first request). Subsequent invoices are faster thanks to prompt caching and the singleton embedder.

What this isn't

A deployed product. The action executor runs in mock mode — every "would have done X" call is logged but no real SAP/Ariba/ServiceNow side effects happen. The architecture is production-shaped (open-source models, structured logging, evaluation harness, HITL queue, audit trail) but the codebase is a learning artifact and reference, not a live service.

The live demo above is for exploration. Don't put real production invoices through it.

Origin

Originally built at TruVs in 2026 while the author (Tribhuvan Joshi) was AI Practice Lead there. It was the reference implementation for the practice's "Multi-System Process Agent" pattern. Now maintained as a personal learning artifact and portfolio piece — the architecture, evaluation discipline, and HITL design are the durable lessons.
